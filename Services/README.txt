Перед первичным запуском:

1. Сделать миграцию во всех сервисах (указать нужные порты для баз данных PostgreSQL и MS SQL). Для сервиса новостей прикреплен файл скрипта для БД для удобства.
2. Запустить все сервисы.
3. Запустить RabbitMQ.
4. Создать в RabbitMQ необходимые очереди (classic, durable):
  AuthUsersQueue - из сервиса авторизации 
  EmployeeRangeForNewsFeed - данные для сервиса новостей
  EmployeeRangeForPersonalAccount - для ЛК
  EmployeeRangeForTimesheet - для таймшита
  RegisterProblemRequestQueue - кляузы пользаков
  RegisterRequestQueue - заявки на регистрацию
и одноименные exchange, тип fanout, привязать к очередям.
5. Через сваггер сервиса авторизации зарегистрировать нового пользователя, указав все необходимые данные
6. Через сваггер EmployeeService поменять у него поле IsAdmin на true, если хотим использовать весь функционал
7. Проверить, что во всех бд создался пользователь с одинаковыми данными
8. Авторизоваться
